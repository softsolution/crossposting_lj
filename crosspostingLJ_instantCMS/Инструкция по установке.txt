Скопируйте файлы данного архива в корень вашего сайта заменяя все встречающееся на пути.3. Если вы используете по умолчанию шаблон для сайта не _default_ - скопируйте файл из архива /templates/_default_/components/com_blog_config.tpl в папку /templates/ваш_шаблон/components.4. Выполните к вашей базе запрос хранящийся в файле query.sql из phpMyAdmin.Изменениы следующие таблицы в базе:cms_blogs - добавлены поля для настроек каждого блогаcms_blogs_posts - добалено поле для синхронизации поста с ЖЖcms_components - дополнительное поле для хранения настроек групп доступа к кросспостингаЕсли у вас таблицы называются по другому - самостоятельно исправьте названия этих таблиц в запросе.ИСПОЛЬЗОВАНИЕ-----------------------------------------------------------1. Включите кросспостинг в настройках компонента блоги в админской части (Компоненты - блоги). Выбирите группы для которых будет доступна функция кросспостинга.2. !!!Выключите плагин Пинг поисковых систем (Пинг и кросспостинг построены на одной библиотеке, что вызывает конфликт. Исправить не удалось - SORRY :-)Где выключить: в админской части - Дополнения - Плагины - Пинг поисковых систем - В колонке 'Включен' должен быть красный кружок с крестиком.3. В настроках блога (с фронтальной части после включения кросспостинга в админке) появится дополнительное поле настроек. (Т.е. для каждого блога в отдельности необходимо включать кросспостинг).4. Включите кросспостинг - и введите логин журнала в ЖЖ и пароль. Также можете поэкспериментировать с другими настройками кросспостинга.5. Сохраните настройки.6. Все готово для использования кросспостинга.ВОЗМОЖНЫЕ КОНФЛИКТЫ-----------------------------------------------------------1. На слабых хостингах возможно увеличение времени публикации поста.
